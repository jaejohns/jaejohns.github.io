How to Use the Liger Grid

Getting Started

Liger’s grid works similar to Bootstrap, Foundation, Skeleton and other grids.

body
<div class=“row”>
</div>
</body>

You must first start with a “row” class.

<div class=“row”>
<div class=“large-4.columns”> </div>
<div class=“large-8.columns”> </div>
</div>

Each row class must be followed by a div class of column. With each column there are two parameters that must be set with the column. 1. Small, medium, and/or large (In this case it is large)
2. Numeric value between 1 - 12. (In this case it is -4 & -8)

Liger’s grid is based on a 12 column width, so each row must have a combined total of 12 columns within that row. The reason Liger uses a 12 column width, as well as most other grids also using a 12 column width is that 12 is easily divided into smaller numbers. 12 can be divided in half, it can be divided in thirds, in fourths, and in sixths.

Notice the above code. It has a row with two sets of columns in it. One column is 4 columns wide and the other is 8 columns wide. Together 4+8 = our 12 columns. So this row will have two columns in it.

<div class=“row”>
<div class=“large-4.columns”> </div>
<div class=“large-4.columns”> </div>
<div class=“large-4.columns”> </div>
</div>

The above row will have 3 columns.

<div class=“row”>
<div class=“large-12.columns”> </div>
</div>

The above row will have 1 column.

Now that the numeric values are covered, let’s move onto the small, medium, large parameter.

Small = Phone
Medium = Tablet
Large = Desktop

Let’s look at the code we had earlier.

<div class=“row”>
<div class=“large-4.columns”> </div>
<div class=“large-8.columns”> </div>
</div>

These two columns are side by side. By default the columns you set are side by side. The 4 columns wide div and the 8 columns wide div will float next to each other.

When we use the parameter small, medium, or large this will determine how these columns will float next to each other when the browser is resized to a phone, tablet, or desktop browser viewing size.

In this case the large-4 and large-8 will be side by side only on a desktop.

<div class=“row”>
<div class=“large-4.medium-4.columns”> </div>
<div class=“large-8.medium-4.columns”> </div>
</div>

If we wanted the large-4 and large-8 columns to float side by side on tablet as well as a desktop we would add the medium parameter.

<div class=“row”>
<div class=“large-4.medium-4.small-4.columns”> </div>
<div class=“large-8.medium-4.small-4.columns”> </div>
</div>

Using the same logical reasoning, we can have the two columns float next to each other on the phone as well.

In these two above examples with medium and the small parameters our columns will follow the same structure from the desktop. With Liger, you have some control to change the structure from desktop to tablet to phone.

<div class=“row”>
<div class=“large-4.medium-6.small-12.columns”> </div>
<div class=“large-8.medium-6.small-12.columns”> </div>
</div>

With the above code, the two column widths are uneven on the desktop. The first column is 1/3 of the total width and the second column is 2/3 thirds of the row width. When we resize the browser to tablet size the two columns change to the same width while still being side by side. They are still side by side because our numeric value is not 12; however when we use the small-12 parameter each column will be on a new line.

The large parameter is for desktop, the medium is tablet, the small is for phones and the hyphenated number is for the width of the column. 2 goes into 12 six times, so -2 would be 1/6 width. 4 goes into 12 three times, so -4 would be 1/3 width. -12 would be full width.

Push, Pull and Offset

Offset

You can move divs up to 11 columns to the right by using classes like .large-offset-1 and .small-offset-3. If you want to move a div to right by 3 column widths, use offset-3. If you want to move a div to right by 11 column widths, use offset-11.

Reordering Your HTML Elements with Push & Pull

Let's say you have an element on the far left side of your website, and then your main content floated to the right of that element. Your website is responsive and when it displays on a phone, that html element is stacked on top of the main content. Well, you don't want that. What if you wanted that html content to be below the main content when viewed on a phone or tablet?

This is the beauty of the Push & Pull attributes. Just by adding a class to your content, the Push & Pull attributes allow you to completely reorder your html content and how they are displayed in the DOM.

.large-push-3, for example, would push a content by 3 columns and this will only be displayed in desktop view because of the "large" prefix in front of "push". .large-push-7 would do the same thing but push the contents 7 columns.

.medium-push-3, for example, would push a content by 3 columns and this will only be displayed in tablet view because of the "medium" prefix in front of "push".

What if you wanted to shift content and only for it to have affect on tablet and phones? Use the small prefix for phone and medium prefix for tablet. The code will look like this class="medium-push-5 small-push-5"

While you push a content, you will want to pull another html content. With the "pull" attribute, you follow the same method as the "push".

Liger's Components & Features

Liger's components and features allow this grid framework to be more powerful than a standalone grid framework. Here is a list of the components & features that make Liger great!

Colors

You don’t have to look up hexadecimal values for colors. Liger comes built in with an assortment of colors you can easily add to your project. There are two ways you can add color to your elements. You can either add color to the typography or the background color of an item.

Colors are categorized as Cool colors, Warm colors, or greyscale.

For font colors all you need to do is type the name of the color as a class, and for background colors you will need to prefix “bg-“ to the color.

<div class=“row”>
<div class=“large-12.medium-12.bg-yellow.black.columns”> </div>
</div>

The above code will add a yellow background color with black text.

Billboard

The billboard is an extremely quick and straightforward way to bring attention to any news, item, product, service, etc you may have to offer.

<div class=“billboard”>
<h1>Hello, world!</h1>
<p>This is a simple hero unit</p>
</div>

Add the billboard class to a parent container that houses images and/or text.

Buttons

<div class=“button”>Learn More</div>
<button>Learn More</button>

Buttons are an essential part of an effective website. Liger’s buttons can be initialized by declaring the button tag or using the button class.

<div class=“button round shadow”>Learn More</div>

You can adjust the shape of the button by adding a round class, circle class, or the full-width class. Also, you can add a shadow to any button with the shadow class.

In addition to changing the shape of a button, Liger’s button component comes with a simple animation when the button the user hovers over the button.

Animated Forms

<div class=“large-12.animated.columns”>
<div class=“input-group”>
<input type="text">
<label>Label Name</label>
</div>
</div>

Liger’s animated form offers a fantastic way to add some life and interactivity to your input fields. By adding the class “animated” to your parent div you can make your input fields animated.

<div class=“large-12.columns”>
<label><input type="radio" name="radio"><span class=“label-text”> </span> </label>
<label><input type="checkbox" name="check"><span class=“label-text”> </span> </label>
</div>

With Liger you can animate radio buttons and checkboxes. By nesting the input type within the label we can easily bring some life to these form elements. You do not need to use the animated class for these two buttons.

Panel

<div class=“panel”>
<h1>Hello, world!</h1>
<p>This is a panel class</p>
</div>

The panel component is a helper class component that adds extra margin and padding around an element. Add the class panel to your parent element.

Pod

<div class=“pod”>
<img src ="your-image-here.jpg''/>
<div class=“caption”>
<h3>Pod label </h3>
<p>Text goes here</p>
</div>
</div>

The pod component allows you to have an image with some text below the image. Add a new div with the class of pod within Liger’s columns to get a great visual display with some essential information.

Media Object

<div class=“media”>
<div class=“--left”>
<img src ="your-image-here.jpg'' class=“--media-object”/>
</div>
<div class=“--body”>
<h4 class=“--heading”>Pod label </h4>
<p>Text goes here</p>
</div>
</div>

Media objects are an essential part of every website. They are combination of a graphic on one side and text next to the graphic. You can see this combination all over the web.

Navbar

<div class=“menu”>
<nav>
<ul>
<li> <a href=“#”>Menu Item 1</a></li>
<li> <a href=“#”>Menu Item 2</a>
<div>
<ul>
<li> <a href=“#”>Menu Item 1</a></li>
</ul>
</div>
</li>
</ul>
</nav>
</div>

A navigation is critical to every website, and sometimes it's a pain to develop or find a responsive navigation. Liger has got you covered! Liger provides a basic dropdown that supports up to three levels of cascading menu items.

It uses an unordered list structure with li's for each item. If you want items to branch out of a list item, place a div before the closing li. Within that div you will have another unordered list with li's. Make sure to encase the nav structure in a div class of menu.

Slider Controls

<div class=“slider-button-wrap”>
<div class=“prev”> <i class=“fa fa-chevron-left fa-3x”> </i> </div>
<div class=“next”> <i class=“fa fa-chevron-left fa-3x”> </i> </div>
</div>

Slider

<div class=“slider”>
<div style=“display: inline-block;'' class=“animated fadeIn”> <img src="your-image-here.jpg''></div>
<div class=“animated fadeIn”> <img src="your-image-here.jpg''></div>
<div class=“animated fadeIn”> <img src="your-image-here.jpg''></div>
</div>

A slider is a great interactive way to feature a particular service or product. You can showcase several items or only a limited few, depending on your needs.

Liger's slider is a full width slider meant to showcase images in all their 100% width glory. When you use Liger's slider componant, you must have the slider controls html then the html for the actual slider. The slider controls use a z-index positioning in order to be placed on top of the images.

Liger's slider componant requires javascript, so make sure to include the javascript file in your project.
