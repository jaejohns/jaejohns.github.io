How to Implement Liger's Components

Getting Started

The Liger CSS Grid Framework has all the components initialized right out of the box. However, you can easily customize the grid to only include the necessary styles you want or need with Liger's @import feature.

When you open the liger.css file, you will notice the below list at the top of the css document.

@import url(https://maxcdn.bootstrapcdn.com/font-awesome/4.2.0/css/font-awesome.min.css);
@import url(navbar.css);
@import url(panel.css);
@import url(billboard.css);
@import url(slider.css);
@import url(pod.css);
@import url(buttons.css);
@import url(animated-form.css);
@import url(media-object.css);
@import url(colors.css);
@import url(typography.css);
@import url(utilities.css);

Each one of these files link to a separate css that helps obtain the functionality of that specific component. If you do not need a specific component; for example, the Slider component, on your website, then feel free to delete the @import line linking to the slider.css file. By deleting a line from the liger css file, you are making this css file size smaller, which is a good thing.

The first file in the list is a link to Font Awesome. Liger uses Font Awesome's CDN in order to aid in some of Liger's animations for its animated form.
